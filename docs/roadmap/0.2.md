Version 0.2
## New Features

- (Main Feature) Support for advanced predicates using expr-lang/expr.
- Support for HTTP/2 H2C (clear-text)
- (mock server) Support for HTTPS server with self signed certificate
- (mock server) Support for HTTP/2 server (also H2C) + possibility to set http2 specific parameters like streaming

##(Main Feature )Support for advanced predicates using expr-lang/expr
- All the fields of the context shoudl be reachable
- We use a dynamic string map to represent the tree-like json stucture
- The expression should be compiled before that the scenarios is started the first time. Make entry in log about this. If compilation fails the scenario is not started and the status is set to "script error"
- The expr statement should evaluate to a boolean. If true we jump to the define next step.
- Be permissive if non-existing variables are called.
- "op" in step is set to "expr" and value is the actual expr string to be compiled
- In manual give an overview of all supported build-in expr function
